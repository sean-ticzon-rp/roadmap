PROMPT ENGINEERING FOR JAVA DEVELOPERS

WHAT IS IT?

Carefully designing prompts/instructions to get better AI outputs. Like giving clear instructions to a teammate.

WHY USE IT?

  • → Prevents hallucinations & bad outputs
  • → Saves money (Claude has usage limits)
  • → Saves time (right answer first try)

WHERE TO USE IT?

  • → Writing code that follows your style
  • → Working with client constraints
  • → PR descriptions & documentation

THE 5 LEVELS OF PROMPTING

PROGRESSIVE APPROACH TO WRITING BETTER PROMPTS

1

NOT EVEN TRYING

Just asking without any context or guidance.

Write me a PR message for "emulator display id and descriptions for item lists under promos tab"

WHAT'S MISSING:

  • ✗ No code context
  • ✗ No format guidance
  • ✗ No understanding of changes
RESULT: GENERIC GARBAGE
2

ADD CONTEXT

Attach your actual files so AI knows what changed.

Write me a PR message for "emulator display id and descriptions for item lists under promos tab" [Attached files:] - PromoService.ts - PromoListComponent.html - PromoListComponent.ts

WHAT IMPROVED:

  • ✓ AI can see actual code changes
  • ✓ More specific, relevant output

STILL MISSING:

  • ✗ Format/structure guidance
  • ✗ Company PR standards
  • ✗ Length limits
RESULT: NEEDS EDITING
3

ADD INSTRUCTIONS & GUARDRAILS

Tell Claude what to do AND what NOT to do.

Write me a PR message for "emulator display id and descriptions for item lists under promos tab" [Attached files:] - PromoService.ts - PromoListComponent.html - PromoListComponent.ts // Instructions (what to DO) - Keep under 200 words - Include: issue resolved, changes made, validation steps - Use bullet points - Technical tone // Guardrails (what NOT to do) - NO emojis - Don't speculate about code not shown - Don't suggest improvements (just describe changes)

WHAT IMPROVED:

  • ✓ Specific format requirements enforced
  • ✓ Company standards applied
  • ✓ Consistent structure every time
BE EXPLICIT

Clear, direct instructions get better results

ADD GUARDRAILS

Tell it what NOT to do (no emojis, no speculation)

GIVE CONTEXT

Explain what the output is for and who reads it

RESULT: DECENT, USABLE
4

ADD EXAMPLES (FEW-SHOT)

Show Claude 3-5 examples of what you want. More examples = better performance.

Write me a PR message for "emulator display id and descriptions for item lists under promos tab" [Attached files and instructions same as Level 3] // Example 1 **Issue:** NullPointerException in RefundService **Changes Made:** - Added null check before accessing discount.code - Updated error handling **Validation:** - Added unit test: testRefundWithNullDiscount() - Tested 5 refunds in staging // Example 2 **Issue:** Timeout in PaymentProcessor **Changes Made:** - Increased timeout from 3s to 10s - Added connection pooling **Validation:** - Load tested 500 concurrent requests - Zero timeouts observed // Example 3 **Issue:** Incorrect tax calculation **Changes Made:** - Fixed tax calculation for multiple addresses - Updated to use destination address **Validation:** - Added integration test - Verified 10 historical orders

WHAT IMPROVED:

  • ✓ Shows AI exactly what "good" looks like
  • ✓ Learns your team's specific style
  • ✓ Consistent format across all PRs

FEW-SHOT PROMPTING: Include 3-5 diverse examples. Claude learns patterns from examples better than from instructions alone.

RESULT: GOOD STYLE
5

XML TAGS FOR STRUCTURE

Use XML tags to organize complex prompts. Claude was trained specifically to recognize XML tags.

Write me a PR message <files_changed> - PromoService.ts - PromoListComponent.html - PromoListComponent.ts </files_changed> <instructions> - Keep under 200 words - Include: issue, changes, validation - Use bullet points, technical tone - NO emojis, don't speculate </instructions> <examples> <example_1> **Issue:** NullPointerException in RefundService **Changes:** Added null check... **Validation:** Unit test added... </example_1> <example_2> [More examples...] </example_2> </examples> <your_task> Write PR for the code changes above </your_task>

WHAT IMPROVED:

  • ✓ Crystal clear structure - no confusion
  • ✓ Claude won't mix examples with instructions
  • ✓ Easy for humans to scan and modify
  • ✓ 30% accuracy improvement (Anthropic data)
WHY XML?

Claude was specifically trained to recognize XML tags as prompt structure

BETTER PARSING

Prevents mixing up different parts of your prompt

HUMAN-READABLE

Both you and AI understand the structure clearly

ROLE PROMPTING

Add <role> tags to adjust Claude's style and expertise

RESULT: PRODUCTION READY

LEVEL TIME TO WRITE OUTPUT QUALITY WHEN TO USE

LEVEL 2 30 seconds NEEDS EDITING Quick code questions
LEVEL 3 2 minutes DECENT Daily tasks, PR descriptions
LEVEL 4 5 minutes GOOD Establishing patterns, team standards
LEVEL 5 5 minutes PRODUCTION READY Complex prompts, automation, reusable templates
